# Внедрение шифрования паролей пользователей

## Проблема
В текущей реализации сервиса - пароли пользователей хранятся в открытом виде в БД, что опасно для данных

## Цель
1. Реализовать шифрование паролей пользователей с помощью секртеного ключа, который будет храниться в .env файле

## Логика работы шифрования

1. Пользователь вводит логин и пароль (уже реализовано)
2. Мы проверяем корректность путем авторизации на сайте (уже реализовано)
3. Мы создаем/изменяем пользователя в БД, а именно:
    1. Шифруем пароль с использованием секрета
    2. Сохраняем в БД
4. Когда нам требуется найти "слоты" для пользователя - мы берем шифрованный пароль из БД, расшифровываем ему и с ним авторизуемся (процесс авторизации на сайтах уже есть, надо реализовать рассшифровку пароля)


## Ограничения
1. Требуется именно шифровать пароль, чтобы его можно было расшифровать. Стандартный подход с хэшированием паролей - НЕЛЬЗЯ
